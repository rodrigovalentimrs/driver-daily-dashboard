🚗 PROJETO — DRIVER CONTROL
Sistema Inteligente para Motoristas de Aplicativo

========================================================
📌 DESCRIÇÃO DO PROJETO
========================================================

Aplicação web para controle financeiro, produtividade
e planejamento de motoristas de aplicativo.

O sistema ajudará o motorista a:

- registrar ganhos
- registrar despesas
- calcular lucro
- acompanhar metas
- analisar desempenho
- entender custo operacional

A proposta NÃO é criar um sistema gigante de uma vez.

O projeto será construído por versões,
começando pelo núcleo principal
e evoluindo gradualmente.

========================================================
🎯 OBJETIVO PRINCIPAL
========================================================

Criar uma aplicação organizada e funcional para:

- prática de front-end
- estudo de arquitetura
- manipulação de tabelas
- cálculos financeiros
- dashboards
- organização de sistemas

O projeto também poderá futuramente evoluir para:
- sistema real
- app mobile
- SaaS
- versão fullstack

========================================================
🧠 CONCEITO DO SISTEMA
========================================================

O sistema será dividido em módulos.

Cada módulo terá uma responsabilidade específica.

O objetivo é evitar:
- excesso de complexidade
- código gigante
- travamentos durante o desenvolvimento

A estratégia será:

1. criar núcleo funcional
2. organizar arquitetura
3. adicionar funcionalidades
4. evoluir gradualmente

========================================================
📄 NÚCLEO PRINCIPAL DO SISTEMA
========================================================

O núcleo principal será:

CONTROLE DIÁRIO FINANCEIRO

Todo o restante do sistema dependerá disso.

========================================================
📄 TELA 1 — CONTROLE DIÁRIO
========================================================

Objetivo:
Registrar os dados diários do motorista.

========================================================
📋 CAMPOS PRINCIPAIS
========================================================

DATA:
- dia da semana
- dia do mês

GANHOS:
- ganho total do dia

DESPESAS:
- combustível
- alimentação
- outras despesas

PRODUTIVIDADE:
- km rodado
- horas trabalhadas

RESULTADOS:
- lucro líquido
- ganho por hora
- ganho por km

========================================================
📊 TOTALIZAÇÃO
========================================================

Ao final da tabela:

- total bruto
- total líquido
- total despesas
- total combustível
- total km
- total horas
- média diária
- média por hora
- média por km

========================================================
🧠 OBSERVAÇÃO IMPORTANTE
========================================================

Inicialmente NÃO separar:

- Uber
- 99
- inDriver
- particular

Motivo:
evitar complexidade excessiva na V1.

Inicialmente:
usar apenas GANHO TOTAL.

Futuramente poderá existir separação por app.

========================================================
📄 TELA 2 — PLANEJAMENTO
========================================================

Objetivo:
Definir metas de faturamento.

========================================================
📋 FUNCIONALIDADES
========================================================

- meta diária
- meta semanal
- meta mensal

O sistema mostrará:

- valor planejado
- valor realizado
- quanto falta para meta

========================================================
📊 DADOS AUTOMÁTICOS
========================================================

Os valores realizados NÃO serão preenchidos manualmente.

O sistema puxará automaticamente os dados
da tela diária.

========================================================
📈 PAINEL DE INDICADORES
========================================================

O sistema poderá mostrar:

- faturamento bruto
- lucro líquido
- despesas
- km rodado
- horas trabalhadas
- média diária
- dias restantes do mês
- valor restante da meta

========================================================
📄 TELA 3 — CUSTO POR KM
========================================================

Objetivo:
Calcular custo operacional do motorista.

========================================================
📋 DADOS PRINCIPAIS
========================================================

- combustível
- consumo do carro
- IPVA
- seguro
- pneus
- óleo
- manutenção
- km mensal

========================================================
📊 RESULTADOS
========================================================

O sistema calculará:

- custo mensal
- custo anual
- custo por km

========================================================
🧠 OBSERVAÇÃO IMPORTANTE
========================================================

Inicialmente NÃO calcular:

- depreciação complexa
- financiamento avançado
- múltiplos cenários

Motivo:
reduzir complexidade inicial.

========================================================
📄 TELA 4 — ANALISADOR DE CORRIDAS
========================================================

Objetivo:
Avaliar corridas manualmente.

========================================================
📋 INPUTS
========================================================

- valor da corrida
- distância
- tempo estimado

========================================================
📊 RESULTADO
========================================================

O sistema mostrará:

- corrida ruim
- corrida aceitável
- corrida boa

Baseado:
- custo por km
- lucro desejado

========================================================
🚀 FUNCIONALIDADES FUTURAS

- app mobile
